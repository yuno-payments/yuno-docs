title: "TransactionConfig"
description: "All fields available in TransactionConfig and what they do."
---

`TransactionConfig` is the configuration object passed to the `Transaction` constructor. It defines the session, locale, appearance, controllers, and behavior flags for a single transaction.

<Note>
On web, `TransactionConfig` is a plain object passed directly to `sdkPayments.transaction(config)`.
</Note>

## Fields

| Field | Type | Required | Default |
|---|---|---|---|
| [`checkoutSession`](#checkoutsession) | String | One of two |  |
| [`customerSession`](#customersession) | String | One of two |  |
| [`countryCode`](#countrycode) | String | Yes |  |
| [`language`](#language) | String? | No | Device/browser language |
| [`appearance`](#appearance) | Appearance? | No | `null` |
| [`controller`](#controller) | TransactionController? | No | `null` |
| [`showPaymentStatusScreen`](#showpaymentstatusscreen) | Bool | No | `false` |
| [`viewController`](#viewcontroller-ios-only) (iOS only) | UIViewController? | No | `null` |

You must pass either `checkoutSession` or `customerSession`, but not both. See [Validation](#validation).

---

## `checkoutSession`

Session ID for a payment flow. Pass the value returned by your backend after calling [`POST /v1/checkout/sessions`](https://docs.y.uno/reference/checkout-sessions/create-checkout-session).

Mutually exclusive with `customerSession`.

See [Sessions](/docs/basic-concepts/sessions) for more.

## `customerSession`

Session ID for an enrollment flow. Pass the value returned by your backend after calling [`POST /v1/customers/sessions`](https://docs.y.uno/reference/customer-sessions-enrollment/create-customer-session).

Mutually exclusive with `checkoutSession`.

See [Sessions](/docs/basic-concepts/sessions) for more.

## `countryCode`

ISO country code for the payment (for example, `US`, `CO`).

Accepts any string. The SDK does not validate the value. See [Country coverage](https://docs.y.uno/resources/country-coverage) for the list of supported countries.

## `language`

ISO-639 language code. Falls back to the device or browser language if omitted.

See [Supported languages](https://docs.y.uno/resources/languages-supported).

## `appearance`

Client-side styling override. When provided, takes precedence over the styling defined in your Checkout Builder.

See [Styling & Appearance](/docs/yuno-sdk/features/styling).

## `controller`

Object that implements the `TransactionController` interface. Used to receive transaction events (`onStatus`, `showView`, `showLoading`, `createPayment`).

On Web, controllers are wired directly as fields on the config (`onStatus`, `showLoading`, `showView`, `createPayment`) or chained on the `Transaction` instance via `transaction.on('status' | 'loading' | 'view', handler)` and `transaction.createPayment(handler)`. On iOS and Android, you pass a single controller object through the `controller` field.

See [Controllers](/docs/yuno-sdk/controllers/overview).

## `showPaymentStatusScreen`

Flag that controls whether the SDK presents its built-in result screen after the transaction finishes.

- `false` (default). The SDK does not show a result screen. You receive the final status through `onStatus` and render your own UI.
- `true`. The SDK presents its result screen.

`onStatus` is always invoked, regardless of this flag.

## `viewController` (iOS only)

Optional reference to the `UIViewController` that the SDK should use to present its UI.

Optional when your app is fully built with SwiftUI. Required if you want the SDK to present from a specific UIKit view controller.

This field exists only on iOS.

---

## Validation

The constructor throws if:

- Both `checkoutSession` and `customerSession` are provided.
- Neither `checkoutSession` nor `customerSession` is provided.

How the error is raised depends on the platform:

| Platform | Error type |
|---|---|
| iOS | `init` throws |
| Android | `IllegalArgumentException` |
| Web | `TypeError` |

The SDK does not validate `countryCode`, `language`, or any other field. Invalid values may surface later as backend errors or unexpected behavior.
