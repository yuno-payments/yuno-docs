title: "PaymentMethodSelected"
description: "The object you pass to start() to tell the SDK which payment method to use."
---

`PaymentMethodSelected` is the object you pass to `Transaction.start()` to tell the SDK which payment method the user picked. You can build it from the result of [Payment Methods List](/docs/yuno-sdk/features/payment-methods-list) or from the payment methods service returned by Yuno's API.

## Fields

| Field | Type | Required | Notes |
|---|---|---|---|
| [`paymentMethodType`](#paymentmethodtype) (mobile) / `type` (web) | String | Yes | |
| [`vaultedToken`](#vaultedtoken) | String? | No | |
| [`transactionData`](#transactiondata) (mobile) / `data` (web) | TransactionData / PaymentMethodData | No | |
| [`metadata`](#metadata) | `Map<String, Any>?` | No | |

Only `paymentMethodType` / `type` is required. The SDK does not validate this object.

---

## `paymentMethodType`

String identifying the type of payment method (for example, `CARD`, `GOOGLE_PAY`, `APPLE_PAY`).

The list of valid values is not fixed. Yuno may add new payment method types over time. Use the value returned by Yuno's API or by the [Payment Methods List](/docs/yuno-sdk/features/payment-methods-list) controller.

## `vaultedToken`

Token of a previously saved payment method, returned by Yuno when the customer enrolled the method or when a previous payment was made with `vault_on_success: true`.

Use this when the customer is paying with a saved method instead of entering data again.

## `transactionData`

Pre-collected payment data. When provided, the SDK skips its built-in forms and tokenizes directly.

See [TransactionData](/docs/yuno-sdk/reference/transaction-data) for the shape of this object.

## `metadata`

Free-form key-value object. Reserved for future use.
