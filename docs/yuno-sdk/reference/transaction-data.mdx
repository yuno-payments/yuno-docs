title: "TransactionData"
description: "Pre-collected payment data passed to the SDK to skip its built-in forms."
---

`TransactionData` is the object you pass inside [`PaymentMethodSelected`](/docs/yuno-sdk/reference/payment-method-selected) when you've already collected the payment data on your side. When provided, the SDK skips its built-in forms and tokenizes directly with the values you supply.

`TransactionData` is the union of several nested objects, each covering a different aspect of the payment (card data, installments, 3DS setup, customer data, etc.). You only fill the parts that apply to your scenario. Every nested object is optional unless required by the specific payment method or feature you're using.

## Shape

The full shape of the object passed to `start()` is the following, with `TransactionData` under `paymentMethod.data`. All field names are camelCase in the SDK. Every field inside `data` is optional — fill only what applies to your scenario.

    "type": "CARD",
    "vaultedToken": "tok_saved_abc",
    "data": {
      "card": {
        "number": "4111111111111111",
        "expirationMonth": 12,
        "expirationYear": 2028,
        "securityCode": "123",
        "holderName": "John Doe",
        "brand": "VISA",
        "type": "CREDIT"
      },
      "vaultOnSuccess": true,
      "installment": {
        "id": "installment_xxx",
        "value": 3
      },
      "threeDSecure": {
        "setupId": "setup_xxx"
      },
      "customer": {
        "firstName": "John",
        "lastName": "Doe",
        "email": "john.doe@example.com",
        "document": {
          "documentNumber": "1090209924",
          "documentType": "CC"
        },
        "phone": {
          "countryCode": "57",
          "number": "3001234567"
        },
        "merchantCustomerId": "customer_xxx",
        "billingAddress": {
          "addressLine1": "Calle 34 # 56 - 78",
          "addressLine2": "Apartamento 502, Torre I",
          "city": "Bogotá",
          "country": "CO",
          "state": "Cundinamarca",
          "zipCode": "111111",
          "neighborhood": "Chapinero"
        },
        "shippingAddress": {
          "addressLine1": "Calle 34 # 56 - 78",
          "addressLine2": "Apartamento 502, Torre I",
          "city": "Bogotá",
          "country": "CO",
          "state": "Cundinamarca",
          "zipCode": "111111"
        }
      },
      "bankTransfer": {
        "bankAccount": "1234567890",
        "routingNumber": "021000021",

      },

        "benefitType": "FOOD"

        "cardData": {

        }

`type` and `vaultedToken` are not part of `TransactionData` — they live on [`PaymentMethodSelected`](/docs/yuno-sdk/reference/payment-method-selected). `viewContainer` (a CSS selector or DOM element, web only) is a sibling of `paymentMethod` on the `start()` argument, not part of `TransactionData` either.

## Top-level objects

| Field | Description |
|---|---|
| `issuerId` | Identifier of the issuer for the payment method, when applicable. |
| `vaultOnSuccess` | When `true`, saves the payment method for the customer if the payment succeeds. |
| `card` | Card data (number, expiration, security code, holder, brand, type). |
| `installment` | Installment plan and value. |
| `threeDSecure` | 3DS setup identifier when 3DS is configured separately. |
| `bankTransfer` | Bank transfer details (beneficiary, account, routing). |
| `wallet` | Wallet-specific card data (e.g., card type). |
| `ticket` | Ticket-based payment data (e.g., benefit type for food/meal vouchers). |
| `otp` | One-time password, when the payment method requires it. |
| `customer` | Customer information (name, email, document, phone, billing and shipping addresses). |
