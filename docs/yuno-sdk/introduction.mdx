Yuno provides client-side SDKs for iOS, Android, and Web. The SDKs share a 
single, unified API: one `Transaction` class, one `start()` method, and a 
small set of configuration parameters and controllers that determine how 
the integration behaves.

You initialize the SDK once with your API key, then create a `Transaction` 
for each payment or enrollment. Controllers and configuration fields are 
optional. By default, the SDK renders forms, tokenizes the payment 
method, creates the payment, handles 3DS, and returns the final status. 
Override controllers or pass additional data when you need more control.

## Choose a session type

The same `Transaction` class handles two flows:

- Pass a **`checkoutSession`** to process a payment.
- Pass a **`customerSession`** to enroll a payment method without 
  charging.

See [Sessions](/docs/basic-concepts/sessions) for details.

## Backend requirements

The SDKs are client-side only. Your backend must:

1. Create checkout or customer sessions.
   - **Checkout session**: see [`POST /v1/checkout/sessions`](https://docs.y.uno/reference/checkout-sessions/create-checkout-session).
   - **Customer session**: see [`POST /v1/customers/sessions`](https://docs.y.uno/reference/customer-sessions-enrollment/create-customer-session).

See [Get Started](/docs/yuno-sdk/get-started) for the full backend 
integration walkthrough.

## Migrating from a previous version

If you're currently using one of the legacy SDKs (Full, Lite, Seamless, 
Render, Headless, Secure Fields), the new unified API replaces all of 
them with a single `Transaction` class. Your existing integration will 
continue to work, but new features and improvements ship in the 
unified SDK.

See the [Migration guide](/docs/yuno-sdk/migration/overview) for a mapping from 
legacy modes to the new API and step-by-step instructions for each 
platform.
