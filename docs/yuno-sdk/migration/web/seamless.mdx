description: "Migrate Web Seamless integrations (yuno.startSeamlessCheckout + yuno.mountSeamlessCheckout / mountSeamlessExternalButtons) to the unified Transaction API."
---

Seamless mode was the **SDK-managed variant**: the SDK created the payment internally, the merchant only chose when/where to render the form and triggered the start via `yuno.startPayment()`. Unlike non-Seamless mode, Seamless did **not** expose `yunoCreatePayment` ŌĆö payment creation was always managed by the SDK.

In the unified API the same flow is a normal `Transaction` **without** `createPayment` wired (presence of `createPayment` flips to manual mode; its absence stays managed). The merchant triggers payment by calling `transaction.start({ paymentMethod })` directly ŌĆö typically from inside `onPaymentSelected` after the user picks a method.

| Before | After |
|---|---|
| `yuno.startSeamlessCheckout(config)` + `yuno.mountSeamlessCheckout()` (Full) | `sdkPayments.transaction(config)` + `getPaymentMethodsViews({ types: ['paymentMethodList'], onPaymentSelected })` + `views.paymentMethodList?.mount({ elementSelector })` |
| `yuno.mountSeamlessExternalButtons([{ elementSelector, paymentMethodType }])` | `getPaymentMethodsViews({ types: ['applePay', 'googlePay', 'paypal'], onPaymentSelected })` + per-view `.mount({ elementSelector, ...config })` |
| `yuno.startPayment()` (merchant trigger) | `transaction.start({ paymentMethod })` |
| Managed payment creation | Managed payment creation (no `createPayment` on config) |

## Side-by-side ŌĆö Seamless Full

<CodeGroup>

```typescript Before. Seamless Full
const yuno = await window.Yuno.initialize('your-public-api-key')

await yuno.startSeamlessCheckout({
    checkoutSession: 'session-from-backend',
    elementSelector: '#root',
    countryCode: 'CO',
    language: 'en',
    yunoPaymentResult: (status) => console.log('done:', status),
})

await yuno.mountSeamlessCheckout()

// Merchant's pay button triggers ŌĆö SDK does everything else
document.getElementById('pay').addEventController('click', () => {
    yuno.startPayment()
})
```

```typescript After. Unified Transaction (managed)
const sdkPayments = await SdkPayments.initialize('your-public-api-key')

const transaction = sdkPayments.transaction({
    checkoutSession: 'session-from-backend',
    countryCode: 'CO',
    language: 'en',
    elementSelector: '#root', // form container for the selected method (form flows only)
    // managed mode ŌĆö no createPayment wired
    onStatus: (result) => console.log('done:', result.status),
})

const views = transaction.getPaymentMethodsViews({
    types: ['paymentMethodList'],
    onPaymentSelected: (paymentMethod) =>
        transaction.start({ paymentMethod }),
})

views.paymentMethodList?.mount({ elementSelector: '#root' })
```

</CodeGroup>

## Side-by-side ŌĆö Seamless External Buttons

<CodeGroup>

```typescript Before. Seamless External Buttons
const yuno = await window.Yuno.initialize('your-public-api-key')

await yuno.startSeamlessCheckout({
    checkoutSession: 'session-from-backend',
    countryCode: 'CO',
    yunoPaymentResult: (status) => console.log('done:', status),
})

yuno.mountSeamlessExternalButtons([
    { elementSelector: '#apple-pay', paymentMethodType: 'APPLE_PAY' },
    { elementSelector: '#google-pay', paymentMethodType: 'GOOGLE_PAY' },
    { elementSelector: '#paypal', paymentMethodType: 'PAYPAL' },
])
```

```typescript After. Unified Transaction (managed, wallet buttons)
const sdkPayments = await SdkPayments.initialize('your-public-api-key')

const transaction = sdkPayments.transaction({
    checkoutSession: 'session-from-backend',
    countryCode: 'CO',
    onStatus: (result) => console.log('done:', result.status),
})

const views = transaction.getPaymentMethodsViews({
    types: ['applePay', 'googlePay', 'paypal'],
    onPaymentSelected: (paymentMethod) => transaction.start({ paymentMethod }),
})

views.applePay?.mount({ elementSelector: '#apple-pay' })
views.googlePay?.mount({ elementSelector: '#google-pay' })
views.paypal?.mount({ elementSelector: '#paypal' })
```

</CodeGroup>

## Field mapping

| Before (Seamless) | After (unified) | Notes |
|---|---|---|
| `yuno.startSeamlessCheckout(config)` | `sdkPayments.transaction(config)` | Single factory. Don't wire `createPayment` ŌĆö that would flip to manual mode (non-Seamless behavior). |
| `yuno.mountSeamlessCheckout()` | `getPaymentMethodsViews({ types: ['paymentMethodList'], onPaymentSelected })` ŌåÆ `views.paymentMethodList?.mount({ elementSelector })` | Get a view handle, then call `.mount(...)` with the selector. |
| `yuno.mountSeamlessExternalButtons([{ elementSelector, paymentMethodType }])` | `getPaymentMethodsViews({ types: ['applePay', 'googlePay', 'paypal'], onPaymentSelected })` ŌåÆ `views.applePay?.mount({ elementSelector, ...config })`, etc. | Per-view styling (button color, type, etc.) lives on each `.mount()` call. |
| `yuno.startPayment()` (merchant trigger) | `transaction.start({ paymentMethod })` | Called from `onPaymentSelected` after the user picks a method. The form renders into the optional `elementSelector` set on `sdkPayments.transaction(config)`. |
| `yunoPaymentResult(status, subStatus?)` | `onStatus(result)` | `PaymentResult` wrapper with `.status` and `.substatus`. |
| `yuno.unmountSeamlessExternalButton(type)` / `yuno.unmountSeamlessExternalButtons()` | *(handled by `transaction.cancel()` or letting the transaction complete)* | Per-button unmount removed. |

## Behavior changes

- **`Seamless` factory is gone.** Use the regular `sdkPayments.transaction(...)` factory. Don't wire `createPayment` ŌĆö its absence keeps the SDK in managed mode (the same model Seamless had).
- **`yuno.startPayment()` is gone.** The merchant-triggered start is now `transaction.start({ paymentMethod })`, called with the method the user picked. When the chosen method renders a form, it uses the optional `elementSelector` set on `sdkPayments.transaction(config)` ŌĆö the list/wallet views keep their own `elementSelector` on `.mount()`.
- **Per-button unmount removed.** Use `transaction.cancel()` or let the transaction complete.
