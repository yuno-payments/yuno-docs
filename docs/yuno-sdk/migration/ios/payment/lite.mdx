title: "Lite mode"
description: "Migrate iOS Lite integrations (Yuno.startPaymentLite) to the unified Transaction API."
---

Lite mode pre-selected the payment method on the merchant side, then handed off to the SDK to render the form, tokenize, create the payment, handle 3DS, and report back. In the unified API this is the **default behavior** of `Transaction.start(paymentSelected:)`. No controller overrides needed.

| Before | After |
|---|---|
| `Yuno.startPaymentLite(with:paymentSelected:showPaymentStatus:)` | `SdkPayments.Transaction(config:).start(paymentSelected:)` |
| `YunoPaymentDelegate` | `SdkPayments.TransactionController` |
| `Yuno.initialize(apiKey:config:)` | `SdkPayments.initialize(apiKey:)` (no config) |

## Side-by-side

<CodeGroup>

```swift Before. Lite mode
import YunoSDK

// 1. Initialize once at app startup
Yuno.initialize(
    apiKey: "your-public-api-key",
    config: YunoConfig(
        appearance: customAppearance,
        saveCardEnabled: true,
        keepLoader: false
    )
)

// 2. Implement the delegate
final class Checkout: YunoPaymentDelegate {
    var checkoutSession: String = "session-from-backend"
    var countryCode: String = "CO"
    var language: String? = "en"
    var viewController: UIViewController?

    func yunoPaymentResult(_ result: Yuno.Result) {
        switch result.status {
        case .succeeded:      print("Success")
        case .reject:         print("Rejected")
        case .fail:           print("Failed")
        case .processing:     print("Processing")
        case .internalError:  print("Internal error")
        case .userCancelled:  print("User cancelled")
        @unknown default:     break
        }
    }

    // 3. Start the payment
    func pay(paymentSelected: PaymentMethodSelected) {
        Yuno.startPaymentLite(
            with: self,
            paymentSelected: paymentSelected,
            showPaymentStatus: true
        )
    }
}
```

```swift After. Unified Transaction
import SdkPayments

// 1. Initialize once at app startup
SdkPayments.initialize(apiKey: "your-public-api-key")

// 2. Implement the controller
final class Checkout: SdkPayments.TransactionController {

    func onStatus(status: SdkPayments.TransactionStatus) {
        switch status.status {
        case "INTERNAL_ERROR":  print("Internal error")
        case "USER_CANCEL":     print("User cancelled")
        // Handle other backend statuses (e.g. SUCCEEDED, DECLINED, PENDING) as needed.
        default:                break
        }
    }

    // 3. Start the payment
    func pay(paymentSelected: SdkPayments.PaymentMethodSelected) {
        let config = SdkPayments.TransactionConfig(
            checkoutSession: "session-from-backend",
            countryCode: "CO",
            language: "en",
            appearance: customAppearance,    // optional
            controller: self,
            showStatusScreen: true    // see "Behavior changes" below
        )

        SdkPayments.Transaction(config: config)
            .start(paymentSelected: paymentSelected)
    }
}
```

</CodeGroup>

## Field mapping

| Before (Lite) | After (unified) | Notes |
|---|---|---|
| `Yuno.initialize(apiKey:config:)` | `SdkPayments.initialize(apiKey:)` | `YunoConfig` is removed. See per-field rows below. |
| `YunoConfig.appearance` | `SdkPayments.TransactionConfig.appearance` | Moves from app-wide to per-transaction. |
| `YunoConfig.saveCardEnabled` | *(removed)* | Now controlled server-side via the settings service. The client no longer passes this flag. |
| `YunoConfig.keepLoader` | *(removed)* | The SDK keeps its loader by default. New: in the unified API you can override [`showLoading`](/docs/yuno-sdk/controllers/show-loading) to render your own. This controller was not invoked in Lite. |
| `YunoPaymentDelegate.checkoutSession` | `SdkPayments.TransactionConfig.checkoutSession` | Now part of the config object, not the delegate. |
| `YunoPaymentDelegate.countryCode` | `SdkPayments.TransactionConfig.countryCode` | Same as above. |
| `YunoPaymentDelegate.language` | `SdkPayments.TransactionConfig.language` | Same as above. |
| `YunoPaymentDelegate.viewController` | `SdkPayments.TransactionConfig.viewController` | Still iOS-only. Optional in SwiftUI; required if you need a specific UIKit presenter. |
| `YunoPaymentDelegate.yunoPaymentResult(_:)` | `SdkPayments.TransactionController.onStatus(status:)` | `status` and `substatus` are passed through from the backend; the SDK only emits `INTERNAL_ERROR` and `USER_CANCEL`. See [`Status`](/docs/yuno-sdk/reference/status). |
| `Yuno.startPaymentLite(with:paymentSelected:showPaymentStatus:)` | `SdkPayments.Transaction(config:).start(paymentSelected:)` | Builder-style. The `showPaymentStatus` argument moved into `SdkPayments.TransactionConfig.showStatusScreen`. |

## Behavior changes

<Warning>
**`showStatusScreen` default flipped.** In Lite, `showPaymentStatus` defaulted to `true`. The SDK presented its built-in result screen. In the unified API, `showStatusScreen` defaults to **`false`**. If your integration relied on the SDK showing the result screen, set `showStatusScreen: true` in `SdkPayments.TransactionConfig` explicitly.
</Warning>

- **`onStatus` is always invoked.** In Lite it was the only way to know the outcome; that hasn't changed. But now it fires regardless of whether the SDK shows its own result screen, so you can drive your own UI on top.
- **Controller retention.** The `controller` field on `SdkPayments.TransactionConfig` follows the same rule the old delegate did: the SDK does not retain it strongly. Keep the instance you pass as `controller` alive for the duration of the transaction.
