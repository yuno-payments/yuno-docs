---
title: "Headless mode"
description: "Migrate iOS Headless integrations (Yuno.apiClientPayment + generateToken) to the unified Transaction API."
---

Headless was a pure-tokenization client: the merchant collected card data in their own UI, called `generateToken(...)`, received a token, and then made their own backend call to create the payment (and optionally triggered 3DS via `getThreeDSecureChallenge()`).

In the unified API the same scenario maps to **non-seamless headless** mode: pass the pre-collected data inside `transactionData` on the `PaymentMethodSelected`, override [`createPayment`](/docs/yuno-sdk/controllers/create-payment) to create the payment from your backend, and let the SDK orchestrate tokenization, 3DS, and status polling automatically. The merchant no longer has to call a separate 3DS endpoint.

| Before | After |
|---|---|
| `Yuno.apiClientPayment(countryCode:checkoutSession:)` → `YunoPaymentHeadless` | `SDKPayment.Transaction(config:).start(paymentSelected:)` |
| `paymentHeadless.generateToken(data:)` | Override `createPayment(token:)` |
| `paymentHeadless.getThreeDSecureChallenge()` | *(removed. SDK handles 3DS automatically)* |
| No delegate | `SDKPayment.TransactionController` |

## Side-by-side

<CodeGroup>

```swift Before. Headless mode
import YunoSDK

// 1. Initialize once at app startup
SDKPayment.initialize(apiKey: "your-public-api-key")

// 2. Create the headless client
let paymentHeadless = try Yuno.apiClientPayment(
    countryCode: "CO",
    checkoutSession: "session-from-backend"
)

// 3. Build the data from your own form
let data = TokenCollectedData(
    checkoutSession: "session-from-backend",
    paymentMethod: CollectedData(
        type: "CARD",
        vaultedToken: nil,
        card: CardData(/* number, securityCode, expiration, holderName */),
        customer: nil
    )
)

// 4. Tokenize, then create the payment on your backend yourself
let result = try await paymentHeadless.generateToken(data: data)
let token = result["token"] as? String ?? ""

await createPaymentOnBackend(token: token)   // your code

// 5. Optional: separately trigger 3DS if your backend asked for it
let challenge = try await paymentHeadless.getThreeDSecureChallenge()
present(HeadlessWebView(url: challenge.url))
```

```swift After. Non-seamless headless
import YunoSDK

// 1. Initialize once at app startup
SDKPayment.initialize(apiKey: "your-public-api-key")

// 2. Implement the controller. Only createPayment + onStatus needed
final class Checkout: SDKPayment.TransactionController {

    func createPayment(token: SDKPayment.OneTimeToken) async throws {
        // Create the payment on your backend with the token.
        // Return when done; the SDK then polls status and runs 3DS if needed.
        try await createPaymentOnBackend(token: token["token"] as? String ?? "")
    }

    func onStatus(status: SdkPayments.TransactionStatus) {
        switch status.status {
        case "INTERNAL_ERROR":  print("Internal error")
        case "USER_CANCEL":     print("User cancelled")
        // Handle other backend statuses (e.g. SUCCEEDED, DECLINED, PENDING) as needed.
        default:                break
        }
    }

    // 3. Build TransactionData from your own form, then start
    func pay(cardNumber: String, cvv: String, expMonth: Int, expYear: Int, holder: String) {
        let paymentSelected = SDKPayment.PaymentMethodSelected(
            paymentMethodType: "CARD",
            transactionData: SDKPayment.TransactionData(
                card: SDKPayment.CardTransactionData(
                    number: cardNumber,
                    securityCode: cvv,
                    expirationMonth: expMonth,
                    expirationYear: expYear,
                    holderName: holder
                )
            )
        )

        let config = SDKPayment.TransactionConfig(
            checkoutSession: "session-from-backend",
            countryCode: "CO",
            controller: self
        )

        SDKPayment.Transaction(config: config)
            .start(paymentSelected: paymentSelected)
    }
}
```

</CodeGroup>

## What changed conceptually

In Headless the merchant did three things explicitly: tokenize, create the payment on their backend, and trigger 3DS. In the unified API only **payment creation** stays on the merchant side. Tokenization and 3DS are handled by the SDK as part of the `Transaction` lifecycle. You signal "payment created, please continue" by returning from `createPayment`; the SDK then queries the payment status and finishes the flow.

If your old integration showed the 3DS challenge inside your own UI (as the example app does with `HeadlessWebView`), you no longer need to. The SDK presents the 3DS WebView full-screen automatically. This is fixed — [`showView`](/docs/yuno-sdk/controllers/show-view) only fires for non-WebView action views (e.g. a QR code), not for 3DS challenges or redirects.

## Field mapping

| Before (Headless) | After (unified) | Notes |
|---|---|---|
| `Yuno.apiClientPayment(countryCode:checkoutSession:)` | `SDKPayment.Transaction(config:)` | The transaction object replaces the headless client. |
| `TokenCollectedData.checkoutSession` | `SDKPayment.TransactionConfig.checkoutSession` | Moves to config. |
| `CollectedData.type` | `SDKPayment.PaymentMethodSelected.paymentMethodType` | Moves to the parent object. |
| `CollectedData.vaultedToken` | `SDKPayment.PaymentMethodSelected.vaultedToken` | Same. |
| `CollectedData.card` (`CardData`) | `SDKPayment.TransactionData.card` (`SDKPayment.CardTransactionData`) | Same fields, namespaced under `transactionData`. |
| `CollectedData.customer` | `SDKPayment.TransactionData.customer` | Moves to `transactionData`. |
| `paymentHeadless.generateToken(data:)` | Override `createPayment(token:)` | The SDK invokes your handler with the token; you create the payment from your backend and return. |
| `paymentHeadless.getThreeDSecureChallenge()` | *(removed)* | The SDK runs 3DS internally and presents the WebView full-screen. Not customizable via `showView`. |
| Result returned from `generateToken` (`[String: Any]`) | `SDKPayment.OneTimeToken` (still a dictionary `[String: Any]`) | Cross-platform parity. See [`OneTimeToken` reference](/docs/yuno-sdk/reference/one-time-token). |
| *(no result controller)* | `onStatus(status:)` | New. The SDK now drives the full lifecycle and reports the final status here. |

## Behavior changes

<Warning>
**3DS is now SDK-driven.** Headless required you to call `getThreeDSecureChallenge()` and present the WebView yourself. The unified API runs 3DS automatically as part of the transaction and presents the WebView full-screen. This is fixed — overriding [`showView`](/docs/yuno-sdk/controllers/show-view) does not change it (showView only fires for non-WebView action views like QR codes).
</Warning>

- **`createPayment` is async.** Your handler is `async throws`. Return when your backend has finished creating the payment. If it throws, the SDK stops the flow and `onStatus` is **not** invoked. Surface the error to your user yourself.
- **No more `apiClient*` objects.** A single `SDKPayment.Transaction` covers the full lifecycle.
- **Controller retention.** Keep the instance you pass as `controller` alive for the duration of the transaction.

- [ ] Replace `TokenCollectedData` / `CollectedData` / `CardData` with `SDKPayment.PaymentMethodSelected` + `SDKPayment.TransactionData` + `SDKPayment.CardTransactionData`.
- [ ] Replace your `paymentHeadless.generateToken(data:)` call with a `SDKPayment.TransactionController.createPayment(token:)` override. Move your "create payment on backend" code there.
- [ ] Implement `onStatus(status:)`. This is now how you receive the final result.
- [ ] Remove `paymentHeadless.getThreeDSecureChallenge()` and the WebView code. The SDK presents 3DS full-screen automatically.
