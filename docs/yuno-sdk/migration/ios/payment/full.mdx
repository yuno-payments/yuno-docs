title: "Full mode"
description: "Migrate iOS Full integrations (Yuno.getPaymentMethodViewAsync + Yuno.startPayment) to the unified Transaction API."
---

Full mode rendered the SDK's payment method list inside your layout, fired callbacks when the user picked a method, then handed off to the SDK to render the form and process the payment. In the unified API the same flow uses a single `Transaction` plus `getPaymentMethodsViews(...)` for the embedded list, with the controllers split in two: `SdkPayments.TransactionController` for the transaction lifecycle and `SdkPayments.PaymentListController` for list-specific events.

| Before | After |
|---|---|
| `Yuno.getPaymentMethodViewAsync(delegate:)` | `transaction.getPaymentMethodsViews(types:controller:)` |
| `Yuno.startPayment(showPaymentStatus:)` | `transaction.start(paymentSelected:)` |
| `YunoPaymentFullDelegate` (single delegate) | `SdkPayments.TransactionController` + `SdkPayments.PaymentListController` (two controllers) |
| `Yuno.initialize(apiKey:config:)` | `SdkPayments.initialize(apiKey:)` |

## Side-by-side

<CodeGroup>

```swift Before. Full mode
import SwiftUI
import YunoSDK

// 1. Initialize once at app startup
Yuno.initialize(
    apiKey: "your-public-api-key",
    config: YunoConfig(appearance: customAppearance, saveCardEnabled: true)
)

// 2. Implement the Full delegate (combines payment + list events)
final class CheckoutModel: YunoPaymentFullDelegate, ObservableObject {
    var checkoutSession: String = "session-from-backend"
    var countryCode: String = "CO"
    var language: String? = "en"
    var viewController: UIViewController?

    @Published var selectedPaymentMethod: PaymentMethodSelected?

    // Payment events
    func yunoPaymentResult(_ result: Yuno.Result) {
        switch result.status {
        case .succeeded:      print("Success")
        case .reject:         print("Rejected")
        case .fail:           print("Failed")
        case .processing:     print("Processing")
        case .internalError:  print("Internal error")
        case .userCancelled:  print("User cancelled")
        @unknown default:     break
        }
    }

    // Payment-list events
    func yunoDidSelect(paymentMethod: PaymentMethodSelected) {
        selectedPaymentMethod = paymentMethod
    }
    func yunoDidUnenrollSuccessfully(_ success: Bool) { /* refresh UI */ }
    func yunoUpdatePaymentMethodsViewHeight(_ height: CGFloat) { /* relayout */ }
}

// 3. Embed the SDK's list and trigger payment
struct CheckoutView: View {
    @StateObject var model = CheckoutModel()
    @State var list: some View?

    var body: some View {
        VStack {
            list
            Button("Pay") {
                Yuno.startPayment(showPaymentStatus: true)
            }
            .disabled(model.selectedPaymentMethod == nil)
        }
        .task {
            list = await Yuno.getPaymentMethodViewAsync(delegate: model)
        }
    }
}
```

```swift After. Unified Transaction
import SwiftUI
import YunoSDK

// 1. Initialize once at app startup
SdkPayments.initialize(apiKey: "your-public-api-key")

// 2. Implement BOTH controllers on the same model (or split them)
final class CheckoutModel:
    SdkPayments.TransactionController,
    SdkPayments.PaymentListController,
    ObservableObject {

    @Published var selectedPaymentMethod: SdkPayments.PaymentMethodSelected?

    // Transaction events
    func onStatus(status: SdkPayments.TransactionStatus) {
        switch status.status {
        case "INTERNAL_ERROR":  print("Internal error")
        case "USER_CANCEL":     print("User cancelled")
        // Handle other backend statuses (e.g. SUCCEEDED, DECLINED, PENDING) as needed.
        default:                break
        }
    }

    // Payment-list events
    func onPaymentSelected(paymentSelected: SdkPayments.PaymentMethodSelected) {
        selectedPaymentMethod = paymentSelected
    }
    func onUnenroll(success: Bool) { /* refresh UI */ }
    func onHeightChange(newHeight: CGFloat) { /* relayout */ }
}

// 3. Build a Transaction, embed the list, trigger payment
struct CheckoutView: View {
    @StateObject var model = CheckoutModel()
    @State var transaction: SdkPayments.Transaction?
    @State var list: AnyView?

    var body: some View {
        VStack {
            list
            Button("Pay") {
                if let selected = model.selectedPaymentMethod {
                    transaction?.start(paymentSelected: selected)
                }
            }
            .disabled(model.selectedPaymentMethod == nil)
        }
        .task {
            let config = SdkPayments.TransactionConfig(
                checkoutSession: "session-from-backend",
                countryCode: "CO",
                language: "en",
                appearance: customAppearance,
                controller: model,
                showPaymentStatusScreen: true
            )
            let tx = SdkPayments.Transaction(config: config)
            transaction = tx
            let views = tx.getPaymentMethodsViews(
                types: [.PAYMENT_METHOD_LIST],
                controller: model
            )
            list = views[.PAYMENT_METHOD_LIST]
        }
    }
}
```

## Field mapping

### Initialization & config

| Before (Full) | After (unified) | Notes |
|---|---|---|
| `Yuno.initialize(apiKey:config:)` | `SdkPayments.initialize(apiKey:)` | `YunoConfig` removed. Per-transaction settings move to `SdkPayments.TransactionConfig`. |
| `YunoConfig.appearance` | `SdkPayments.TransactionConfig.appearance` | Per-transaction now. |
| `YunoConfig.saveCardEnabled` | *(removed)* | Controlled server-side. |
| `YunoConfig.keepLoader` | *(removed)* | SDK keeps the loader by default. Override [`showLoading`](/docs/yuno-sdk/controllers/show-loading) to opt out. |
| `YunoPaymentFullDelegate.checkoutSession` | `SdkPayments.TransactionConfig.checkoutSession` | Moves from delegate to config. |
| `YunoPaymentFullDelegate.countryCode` | `SdkPayments.TransactionConfig.countryCode` | Same. |
| `YunoPaymentFullDelegate.language` | `SdkPayments.TransactionConfig.language` | Same. |
| `YunoPaymentFullDelegate.viewController` | `SdkPayments.TransactionConfig.viewController` | Still iOS-only. |

### Transaction events (`SdkPayments.TransactionController`)

| Before `YunoPaymentFullDelegate` (inherited) | After `SdkPayments.TransactionController` | Notes |
|---|---|---|
| `yunoPaymentResult(_:)` | `onStatus(status:)` | `status` and `substatus` are passed through from the backend. See [`Status`](/docs/yuno-sdk/reference/status). |

### Payment-list events (`SdkPayments.PaymentListController`)

| Before `YunoPaymentFullDelegate` | After `SdkPayments.PaymentListController` | Notes |
|---|---|---|
| `yunoDidSelect(paymentMethod:)` | `onPaymentSelected(paymentSelected:)` | Required. |
| `yunoDidUnenrollSuccessfully(_:)` | `onUnenroll(success:)` | Optional. Same `Bool` payload. |
| `yunoUpdatePaymentMethodsViewHeight(_:)` | `onHeightChange(newHeight:)` | Optional. |

### Entry points

| Before (Full) | After (unified) | Notes |
|---|---|---|
| `Yuno.getPaymentMethodViewAsync(delegate:) async -> some View` | `transaction.getPaymentMethodsViews(types:controller:) -> [SdkPayments.PaymentMethodType: AnyView]` | Returns a dictionary keyed by type. Pass `[.PAYMENT_METHOD_LIST]` for the equivalent of Full. You can also request `.APPLE_PAY` in the same call to get the wallet button. |
| `Yuno.startPayment(showPaymentStatus:)` | `transaction.start(paymentSelected:)` | The SDK no longer remembers the selection internally. Pass it explicitly from your model. The `showPaymentStatus` argument moved to `SdkPayments.TransactionConfig.showPaymentStatusScreen`. |

## Behavior changes

<Warning>
**`showPaymentStatusScreen` default flipped.** In Full, `showPaymentStatus` defaulted to `true`. The SDK presented its built-in result screen. In the unified API, `showPaymentStatusScreen` defaults to **`false`**. If you relied on the SDK showing the result screen, set `showPaymentStatusScreen: true` in `SdkPayments.TransactionConfig` explicitly.
</Warning>

- **Two controllers instead of one.** In Full, `YunoPaymentFullDelegate` mixed transaction events (`yunoPaymentResult`) and list events (`yunoDidSelect`, etc.). In the unified API these are split. You can still implement both on the same class. See the example above.
- **`start()` requires the selection.** Full's `Yuno.startPayment()` looked up the last selection internally. `transaction.start(paymentSelected:)` requires you to pass the `SdkPayments.PaymentMethodSelected` you stored from `onPaymentSelected`.
- **List view return type changed.** `getPaymentMethodViewAsync` returned a single `some View`. The new `getPaymentMethodsViews(types:controller:)` returns `[SdkPayments.PaymentMethodType: AnyView]`. Read `views[.PAYMENT_METHOD_LIST]` â€” the subscript returns `nil` for types that do not apply on iOS.
- **Controller retention.** As with the old delegate, the SDK does not retain the controller strongly. Keep the instance you pass as `controller` alive for the duration of the transaction.

- [ ] Replace `Yuno.initialize(apiKey:config:)` with `SdkPayments.initialize(apiKey:)`.
- [ ] Move `appearance` into `SdkPayments.TransactionConfig`. Drop `saveCardEnabled` and `keepLoader`.
- [ ] Replace conformance to `YunoPaymentFullDelegate` with `SdkPayments.TransactionController` + `SdkPayments.PaymentListController` (same class can implement both).
- [ ] Move `checkoutSession`, `countryCode`, `language`, `viewController` from delegate properties into `SdkPayments.TransactionConfig`.
- [ ] Rename `yunoPaymentResult(_:)` to `onStatus(status:)` and switch the enum cases.
- [ ] Rename `yunoDidSelect(paymentMethod:)` to `onPaymentSelected(paymentSelected:)`.
- [ ] Rename `yunoDidUnenrollSuccessfully(_:)` to `onUnenroll(success:)`.
- [ ] Rename `yunoUpdatePaymentMethodsViewHeight(_:)` to `onHeightChange(newHeight:)`.
