---
title: "Seamless mode"
description: "Migrate iOS Seamless integrations (Yuno.startPaymentSeamlessLite) to the unified Transaction API."
---

Seamless was the simplest legacy entry point: a single `await` call that returned the final result. No delegate, no callbacks. You handed in a `SeamlessParams` and a `PaymentMethodSelected`; the SDK rendered the form, tokenized, created the payment, ran 3DS, and returned a `Yuno.Result`.

In the unified API the same flow is the **default behavior** of `Transaction.start(paymentSelected:)`. No controller overrides needed beyond `onStatus`. The only structural change is the result-delivery model: was async-return, now is a callback through `onStatus`.

| Before | After |
|---|---|
| `Yuno.startPaymentSeamlessLite(with:paymentSelected:showPaymentStatus:)` async → `Yuno.Result` | `SDKPayment.Transaction(config:).start(paymentSelected:)` + `onStatus` controller |
| `SeamlessParams` | `SDKPayment.TransactionConfig` |
| Result returned from `await` | Delivered through `onStatus(status:)` |
| No delegate | `SDKPayment.TransactionController` (just for `onStatus`) |

## Side-by-side

<CodeGroup>

```swift Before. Seamless mode
import YunoSDK

// 1. Initialize once at app startup
SDKPayment.initialize(apiKey: "your-public-api-key")

// 2. Trigger the payment from a single async call
func pay(paymentSelected: PaymentMethodSelected) async {
    let params = SeamlessParams(
        checkoutSession: "session-from-backend",
        countryCode: "CO",
        language: "en",
        viewController: nil
    )

    let result = await Yuno.startPaymentSeamlessLite(
        with: params,
        paymentSelected: paymentSelected,
        showPaymentStatus: true
    )

    switch result.status {
    case .succeeded:      print("Success")
    case .reject:         print("Rejected")
    case .fail:           print("Failed")
    case .processing:     print("Processing")
    case .internalError:  print("Internal error")
    case .userCancelled:  print("User cancelled")
    @unknown default:     break
    }
}
```

```swift After. Unified Transaction
import YunoSDK

// 1. Initialize once at app startup
SDKPayment.initialize(apiKey: "your-public-api-key")

// 2. Implement a minimal controller
final class Checkout: SDKPayment.TransactionController {

    func onStatus(status: SDKPayment.PaymentStatus) {
        switch status.value {
        case .INTERNAL_ERROR:  print("Internal error")
        case .USER_CANCELLED:  print("User cancelled")
        // Handle other backend statuses (e.g. SUCCEEDED, REJECT, FAIL, PROCESSING) as needed.
        @unknown default:      break
        }
    }

    // 3. Trigger the payment
    func pay(paymentSelected: SDKPayment.PaymentMethodSelected) {
        let config = SDKPayment.TransactionConfig(
            checkoutSession: "session-from-backend",
            countryCode: "CO",
            language: "en",
            viewController: nil,
            controller: self,
            showPaymentStatusScreen: true
        )

        SDKPayment.Transaction(config: config)
            .start(paymentSelected: paymentSelected)
    }
}
```

</CodeGroup>

## Field mapping

| Before (Seamless) | After (unified) | Notes |
|---|---|---|
| `SeamlessParams.checkoutSession` | `SDKPayment.TransactionConfig.checkoutSession` | Same. |
| `SeamlessParams.countryCode` | `SDKPayment.TransactionConfig.countryCode` | Same. |
| `SeamlessParams.language` | `SDKPayment.TransactionConfig.language` | Same. |
| `SeamlessParams.viewController` | `SDKPayment.TransactionConfig.viewController` | Still iOS-only. |
| `Yuno.startPaymentSeamlessLite(with:paymentSelected:showPaymentStatus:)` async | `SDKPayment.Transaction(config:).start(paymentSelected:)` | Builder-style. The `showPaymentStatus` argument moved into `SDKPayment.TransactionConfig.showPaymentStatusScreen`. |
| Returned `Yuno.Result` | `SDKPayment.PaymentStatus` delivered through `onStatus` | `status` and `substatus` are passed through from the backend. See [`Status`](/docs/yuno-sdk/reference/status). |

## Behavior changes

<Warning>
**Result is now callback-based, not async-return.** Seamless returned `Yuno.Result` from an `await` call. The unified API delivers the result through the `onStatus` controller. You cannot `await` it inline anymore. If your code is structured around the awaited return value (chained logic, returning to caller), you'll need to refactor to react inside `onStatus`.
</Warning>

<Warning>
**`showPaymentStatusScreen` default flipped.** In Seamless, `showPaymentStatus` defaulted to `true`. In the unified API, `showPaymentStatusScreen` defaults to **`false`**. Set it to `true` in `SDKPayment.TransactionConfig` if you relied on the SDK's built-in result screen.
</Warning>

- **You now need a controller instance.** Even for the simplest case you implement a `SDKPayment.TransactionController` (just `onStatus`). The SDK does not retain it strongly. Keep it alive for the duration of the transaction.

## Checklist

- [ ] Implement `SDKPayment.TransactionController` on a class you can keep alive (e.g. your view model).
- [ ] Move `SeamlessParams` fields (`checkoutSession`, `countryCode`, `language`, `viewController`) into `SDKPayment.TransactionConfig`.
- [ ] Replace the `await Yuno.startPaymentSeamlessLite(...)` call with `SDKPayment.Transaction(config:).start(paymentSelected:)`.
- [ ] Move the result-handling logic from after the `await` into `onStatus(status:)`. Switch the enum cases to UPPER_CASE.
- [ ] Set `showPaymentStatusScreen: true` if you relied on the SDK showing the result screen.
- [ ] Re-run your sandbox test suite.
