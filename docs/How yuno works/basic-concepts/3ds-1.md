## What is 3D Secure

3D Secure (3DS) is an additional security layer that protects card-not-present (CNP) transactions. First released in 1999, it adds an extra step to verify the shopper before a payment is approved, helping issuers stop fraudulent purchases. The diagram below shows a typical 3DS flow:

![3DS Payment Process](https://raw.githubusercontent.com/writechoiceorg/yuno-images/main/doc/concepts/concepts-3ds.png)

## Key components

* **Merchant Plugin Interface (MPI)** – starts the verification and shares encrypted data between the merchant, the card network, and the issuer.
* **Directory Server (DS)** – managed by the card network, it routes the request to the correct issuer and helps determine the best authentication method.
* **Access Control Server (ACS)** – owned by the issuer, it validates the cardholder based on bank policies and risk checks.

## 3D Secure 2 (3DS2)

Released in 2016, 3D Secure 2 (3DS2) upgrades the protocol with better user experiences and stronger authentication. Instead of static passwords, issuers can now rely on biometrics, push notifications, and one-time passcodes to confirm the cardholder. EMVCo manages the standard, and since October 2022 major card brands have retired the original 3DS, so 3DS2 support is now mandatory. Yuno natively supports 3DS2 to help you launch secure, low-friction payments faster.

## Benefits of 3D Secure 2

3DS2 enhances security while meeting modern UX expectations. The protocol focuses on three core themes:

* **Mobile-first experiences** – issuers can prompt for FaceID, TouchID, or other biometric checks inside their own apps, keeping the flow familiar for shoppers.
* **Native integrations** – SDKs allow you to keep users inside your web or mobile checkout instead of redirecting to an external page.
* **Richer data sharing** – merchants can send more contextual information (device IDs, shipping details, historical behavior) so issuers can approve low-risk transactions without added steps.

### Authentication flows in 3DS2

#### Frictionless flow

The issuer silently authenticates the cardholder using background data and approves the payment without showing a prompt to the shopper.

#### Challenge flow

When risk is higher or data is incomplete, the issuer asks the shopper for another factor—usually a biometric confirmation, one-time passcode, or app push notification. The goal is to confirm the cardholder while keeping the experience as smooth as possible.

3DS2 strikes a balance between security and conversion: it limits fraud losses for issuers and merchants while keeping most trusted shoppers in a frictionless path.***
