title: "Webhooks Overview"
description: "Explains what a Yuno webhook is, how delivery and retries work, and where to configure one"
---

A webhook is how Yuno tells your system about events, like a payment authorized, a subscription renewed, or a payout completed, without you having to poll for them. You configure a destination `url`, the events you want (triggers), and how Yuno authenticates to your endpoint. Yuno then delivers each event as an HTTP `POST` as soon as it happens.

To receive webhooks, your endpoint must be a public HTTPS URL that accepts an unauthenticated `POST` request. This is safe because the URL you configure is only ever called by Yuno, and you can verify every delivery is genuine with one of the [authentication methods](/reference/webhooks#delivery-authentication) below.

## Delivery and retries

Yuno expects an **HTTP 200 OK** response to confirm a notification was received. No response body is required. If it doesn't get one in time, Yuno retries up to seven times on the schedule below, so a brief outage on your end doesn't lose an event.

| Attempt | Deadline after the first try |
| :------ | :---------------------------- |
| First try | N/A |
| Second try | 5 minutes |
| Third try | 50 minutes |
| Fourth try | 6 hours |
| Fifth try | 24 hours |
| Sixth try | 48 hours |
| Seventh try | 96 hours |
