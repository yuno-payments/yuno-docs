---
title: "Lite mode"
description: "Migrate Web Lite integrations (yuno.startCheckout + yuno.mountCheckoutLite) to the unified Transaction API."
---

Lite mode pre-selected the payment method on the merchant side, then handed off to the SDK to render the form for that method, tokenize, create the payment, handle 3DS, and report back. In the unified API this is the **default behavior** of `transaction.start({ paymentMethod })`. No controller overrides beyond `onStatus` needed.

| Before | After |
|---|---|
| `yuno.startCheckout(config)` + `yuno.mountCheckoutLite({ paymentMethodType, vaultedToken })` | `sdkPayments.transaction(config).start({ paymentMethod })` |
| Controller fields on `startCheckout` config (`yunoPaymentResult`, `yunoCreatePayment`, `yunoError`, `onLoading`, `onRendered`, ŌĆ”) | `onStatus`, `showLoading`, `showView`, `createPayment` on `TransactionConfig` |
| `Yuno.initialize(publicApiKey)` returning the instance via promise | `SdkPayments.initialize(publicApiKey)` returning `SdkPayments` |

## Side-by-side

<CodeGroup>

```typescript Before. Lite mode
// 1. Initialize once at page startup
const yuno = await window.Yuno.initialize('your-public-api-key')

// 2. Configure the checkout
await yuno.startCheckout({
    checkoutSession: 'session-from-backend',
    elementSelector: '#root',
    countryCode: 'CO',
    language: 'en',
    renderMode: { type: 'modal' },
    yunoPaymentResult: (status) => {
        console.log('Final status:', status)
    },
    yunoError: (error) => {
        console.log('Error', error)
    },
})

// 3. Mount Lite for the pre-selected payment method
await yuno.mountCheckoutLite({
    paymentMethodType: 'PIX',
    vaultedToken: null,
})
```

```typescript After. Unified Transaction
// 1. Initialize once at page startup
const sdkPayments = await SdkPayments.initialize('your-public-api-key')

// 2. Build the transaction and start with the pre-selected method
const transaction = sdkPayments.transaction({
    checkoutSession: 'session-from-backend',
    countryCode: 'CO',
    language: 'en',
    onStatus: (result) => {
        console.log('Final status:', result.status)
    },
})

await transaction.start({
    paymentMethod: { type: 'PIX' },
})
```

</CodeGroup>

## Field mapping

| Before (Lite) | After (unified) | Notes |
|---|---|---|
| `window.Yuno.initialize(...)` | `SdkPayments.initialize(...)` | Global renamed for clarity. |
| `yuno.startCheckout({ checkoutSession, countryCode, language, ŌĆ” })` | `sdkPayments.transaction({ checkoutSession, countryCode, language, ŌĆ” })` | Same fields, single factory. |
| `elementSelector` (where to mount the SDK) | *(removed at session level)* | Per-view mounting moved to `getPaymentMethodsViews({ ŌĆ” })` with each view's `elementSelector`. For Lite (pre-selected method), no mounting is needed ŌĆö `start()` shows the form modally. |
| `renderMode: { type: 'modal' \| 'element', elementSelector }` | `showView` controller (when embedding) | If you set `renderMode: { type: 'element' }` to embed the form, override [`showView`](/docs/universal-sdk/controllers/show-view) instead. Default is modal. |
| `card: CardConfig` | `card: CardConfig` | Same shape. |
| `yunoCreatePayment(ott, info)` | `createPayment(token) => Promise<void>` | Promise-based, single argument (full `OneTimeToken`). |
| `yunoPaymentResult(status, subStatus?)` | `onStatus(result)` | Single `PaymentResult` wrapper. |
| `yunoError(message, data?)` | *(removed)* | Errors flow through `onStatus` (`PaymentResult.status` and `StartResult.error`). |
| `onLoading({ isLoading, type })` | `showLoading(isLoading)` | Drops the deprecated `type` field. |
| `onRendered()` / `onCreateOneTimeToken()` | *(removed)* | Not exposed in the unified API. |
| `yuno.mountCheckoutLite({ paymentMethodType, vaultedToken })` | `transaction.start({ paymentMethod: { type: paymentMethodType, vaultedToken } })` | The Lite pre-selection becomes the `paymentMethod` argument of `start()`. |
| `yuno.continuePayment()` | *(automatic)* | The SDK orchestrates 3DS internally; no manual continue call. |

### Result mapping

The result shape is now wrapped ŌĆö `onStatus` receives a `PaymentResult` instead of two positional arguments.

| Before `yunoPaymentResult` args | After `PaymentResult` fields |
|---|---|
| `status: 'SUCCEEDED' \| 'REJECT' \| 'FAIL' \| 'PROCESSING' \| 'READY'` | `result.status` (same literal union) |
| `subStatus: string?` | `result.substatus` |

## Behavior changes

- **`continuePayment` is internal.** Lite required calling `yuno.continuePayment()` from inside `yunoCreatePayment` to advance the flow. The unified API orchestrates 3DS and status polling automatically when `createPayment` resolves.
- **`yunoError` removed.** All terminal outcomes flow through `onStatus` (or `StartResult.error` from the `start()` Promise).
- **`elementSelector` removed at session level.** Mounting points are per-view (see [`getPaymentMethodsViews`](/docs/universal-sdk/features/payment-methods-list)) or via `showView` for embedded forms.
- **Global rename.** `window.Yuno` ŌåÆ `window.SdkPayments`.

- [ ] Rename `window.Yuno.initialize(...)` to `SdkPayments.initialize(...)`.
- [ ] Replace `yuno.startCheckout(...)` + `yuno.mountCheckoutLite(...)` with a single `sdkPayments.transaction(...).start({ paymentMethod })` call.
- [ ] Move `checkoutSession`, `countryCode`, `language`, `card`, etc. into `TransactionConfig`.
- [ ] Replace `yunoPaymentResult` with `onStatus` and read `result.status` / `result.substatus`.
