The unified SDK replaces all previous SDK modes — Full, Lite, Seamless, Render, Headless, and Secure Fields — with a single `Transaction` class. If you're currently using any of those modes, pick your platform and legacy mode from the table below.

## What changed

Every legacy mode maps to the same `Transaction` class. The mode name is gone — what you pass to the config and which controllers you override determines the behavior:

- **Session type** (`checkoutSession` vs `customerSession`) determines payment vs enrollment.
- **`createPayment` controller** determines whether the SDK creates the payment or your backend does. If you don't override it, the SDK manages payment creation end-to-end (the Seamless equivalent).
- **`showView` controller** determines whether the SDK presents the form full-screen or hands you the view to embed (the Render equivalent).
- **`transactionData` on `PaymentMethodSelected`** determines whether the SDK shows a form or tokenizes data you've already collected (the Headless equivalent).
- **`getPaymentMethodsViews`** replaces all legacy list-mounting and external wallet button APIs (the Full equivalent).

Each guide includes a side-by-side code comparison, a field-mapping table, a behavior-change section, and a migration checklist.
