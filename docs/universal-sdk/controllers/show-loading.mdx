---
title: "showLoading"
description: "Invoked when the SDK enters or exits a loading state."
---

`showLoading` is invoked when the SDK enters or exits a loading state. By default, the SDK shows its own full-screen loader; override this controller to render your own.

## Behavior

If you do not override this controller, the SDK shows a full-screen loader during tokenization, payment creation, 3DS, and status polling.

When you override it, the SDK does not show a loader of its own. Instead, it emits `isLoading: true` when entering a loading state and `isLoading: false` when leaving it. You render your own loader based on those values.

The controller can be invoked multiple times during a single transaction as the SDK enters and exits loading states.

## Signature

<Tabs>
  <Tab title="Web">

```typescript
sdkPayments.transaction({
  ...,
  showLoading: (isLoading) => mySpinner.toggle(isLoading),
})

// or chained
sdkPayments.transaction({...}).on('loading', (isLoading) => mySpinner.toggle(isLoading))
```

  </Tab>

  <Tab title="iOS">

```swift
func showLoading(isLoading: Bool)
```

  </Tab>

  <Tab title="Android">

```kotlin
fun showLoading(isLoading: Boolean)
```

  </Tab>
</Tabs>

## Threading

On iOS and Android, `showLoading` is invoked on the main thread. You can update UI directly from inside this handler.
