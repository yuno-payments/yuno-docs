This page walks you through every step needed to start accepting payments with Yuno on your VTEX store. The setup happens in two places: your **VTEX dashboard** (to register Yuno as a payment provider, install the Payment App, and activate payment methods) and your **Yuno dashboard** (to configure the webhook that keeps order status in sync).

### Before you start

Have the following ready:
* A user with administrator access to your VTEX dashboard.
* Your Yuno credentials, which you can find in your Yuno dashboard under **Developers -> API Credentials**: Account ID, Public API Key, and Private Secret Key.

---

## Step 1 — Configure Yuno as a payment provider

1. In the **VTEX dashboard**, open the left-hand menu and go to **Store Settings -> Payment -> Providers**, then click **New Provider** at the top right.
2. In the **Select a provider** dialog, type **Yuno** in the search box and select **Yuno** from the results.
3. Fill in the provider configuration form using the field reference below, then click **Save**.

### Field reference

| Field | Required | What it does |
| :--- | :--- | :--- |
| **Application Key / Application Token** | Yes | VTEX admin credentials the plugin uses to interact with VTEX on your behalf. They must belong to a user with admin-level access. |
| **Automatic Settlement** | Yes | Defines when the payment capture happens automatically after authorization. See the [Settlement Options](#settlement-options) section for details. |
| **Affiliation Name** | Yes | A human-readable name for this Yuno integration. Used when configuring payment methods to define which affiliation a given payment method should be processed by. |
| **Account ID** | Yes | Your Yuno `account_id`. |
| **Public API Key** | Yes | Your Yuno `public-api-key`. The key's prefix determines the Yuno environment (see note below). |
| **Private Secret Key** | Yes | Your Yuno `private-secret-key`. |
| **Main Account Name** | Optional | For multi-account setups. The name of the main VTEX account to use as the reference for catalog and order lookups. |
| **Main Account App key** | Optional | Application Key of the main VTEX account. Required if Main Account Name is set. |
| **Main Account App token** | Optional | Application Token of the main VTEX account. Required if Main Account Name is set. |
| **Soft Descriptor** | Optional | Custom text that appears on the shopper's card statement. Helps reduce chargebacks related to unrecognized transactions. |
| **Create Customer** | Optional (Default **No**) | When set to **Yes**, the plugin creates a customer record in Yuno for each shopper. Required for saved-card flows and Click to Pay. |
| **VTEX IO orderPlaced page URL** | Optional | A custom URL where shoppers should be redirected after a successful payment. Helps prevent [Known Issue #786685](https://help.vtex.com/known-issues/shopper-is-redirect-to-gatewaycallbackordergroupsuccess-instead-of-placeorder-page). |
| **Payment Mode** | Optional | **Payment App** (modern, recommended) uses the embedded checkout. **Legacy** uses a redirect-based flow. |
| **Transaction Identification From** | Optional (Default **Yuno TID**) | Determines which ID is reported to VTEX: **Yuno TID** or **Provider TID** (useful for reconciliation against acquirer statements). |

<Note>
**Environments**
The Yuno environment your store connects to is determined by the prefix of your Public API Key: `dev_`, `staging_`, `sandbox_`, or `prod_`. Use the production key for your live store and a sandbox key for testing. There is no separate environment switch.
</Note>

### Settlement Options

The **Automatic Settlement** field determines the timing of the capture for card and wallet payments. 

| Option | Practical Behavior |
| :--- | :--- |
| **Use behavior recommended by payment processor (default)** | **Immediate Capture**: The transaction is captured immediately after authorization. |
| **Automatic capture immediately after payment authorization** | **Immediate Capture**: Same as default. |
| **Automatic capture immediately after anti-fraud analysis** | **Immediate Capture**: Since the connector does not introduce a separate wait step, it behaves the same as immediate capture. |
| **Disabled** | **Authorize Only**: The transaction is only authorized. Capture happens later when the order is **invoiced** in VTEX. Recommended for stores that need to verify stock before capturing funds. |

<Note>
**Push Payments**
Methods like **PIX** and **Boleto** are not influenced by this setting, as they are push payments without a separate authorization/capture step.
</Note>

## Step 2 — Install the Yuno Payment App

After saving the provider configuration, install the Yuno Payment App from your VTEX dashboard.

1. In the **VTEX dashboard**, go to **Apps -> App Management**.
2. Search for **Yuno** and select the **Yuno Payment App** in the list of available apps.
3. Click **Install** to complete the setup.

Once installed, the Payment App takes care of rendering the Yuno checkout experience inside your VTEX checkout. There is nothing else to configure on the app itself.

---

## Step 3 — Activate payment methods

Now choose which payment methods you want to offer to your shoppers and route them through Yuno.

1. In the **VTEX dashboard**, go to **Store Settings -> Payment -> Settings -> Payment Conditions**. Click the **+** button to add a payment condition.
2. Select the payment method you want to activate (e.g., Visa, Mastercard, Pix, Apple Pay).
3. In the **Process with provider** field, select the Yuno affiliation you configured in Step 1.
4. Set the **Status** to **Active** and click **Save**.

<Note>
Repeat these steps for every method you want to offer.
</Note>

---

## Step 4 — Configure the webhook in your Yuno dashboard

The webhook is what keeps your VTEX orders in sync with Yuno's payment status updates. You configure it once, in the Yuno dashboard.

1. In the **Yuno dashboard**, open **Developers -> Webhooks** and click **Add webhook**.
2. Fill in the form:
    - **Name**: Any descriptive name, for example `VTEX webhook`.
    - **Endpoint URL**: `https://{store_name}.myvtex.com/_v/yunopartnerbr.yuno/v4/webhook` (replace `{store_name}`).
    - **x-api-key** and **x-secret**: These are required by the form but NOT used by the VTEX plugin. Enter any placeholder value (e.g., `VTEX` in both).
3. Select the events you want Yuno to notify your store about. We recommend enabling all available events to keep order statuses fully in sync.
4. Click **Add**.

## Multi-account and multi-affiliation

These options apply only to specific scenarios. If neither describes your setup, you can skip this section.

### Multi-affiliation
A single VTEX account can be connected to **more than one Yuno integration** when you need to process different payment methods through different Yuno accounts. For example, you can assign one Yuno account to credit cards and a different one to wallets.

To set this up, repeat Step 1 once per integration, giving each one a distinct **Affiliation Name**. Then, when activating payment methods, use the **Process with affiliation** field to choose the correct account.

### Multi-account
If your business runs **multiple VTEX accounts** (for instance, a franchise model sharing the same Yuno credentials), use the **Main Account** fields in the provider configuration:
- **Main Account Name**: The name of the main VTEX account.
- **Main Account App key** and **Main Account App token**: Application credentials of that main account.

When these fields are set, the plugin reads catalog and order data from the main account instead of the current one.
