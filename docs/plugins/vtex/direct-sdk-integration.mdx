---
title: "Direct SDK integration"
description: "Render the Yuno checkout experience directly on your store's front-end for complete control."
---

Most stores integrate Yuno through the standard plugin flow. However, for stores that need to render the Yuno checkout experience directly on their own front-end, Yuno provides the **Yuno SDK Web for VTEX** as an npm package.

## When this applies

Use the direct SDK integration only when your store has specific needs that require the Yuno checkout experience to be rendered directly by your front-end. If your requirements are standard, the [standard plugin flow](/docs/plugins/vtex/set-up-yuno-on-vtex) already handles everything.

### What stays the same
* You still configure Yuno as a payment provider in the VTEX dashboard.
* You still configure the webhook in the Yuno dashboard.
* The **Yuno Payment Connector** remains the backbone of the integration — every payment continues to flow through it.

### What's different
Instead of letting the standard flow render the checkout for you, your front-end mounts the **Yuno SDK Web for VTEX** directly and provides it with the configuration it needs.

---

## Installation

Install the SDK in your project using your preferred package manager:

```bash
npm install @yuno-payments/sdk-web-vtex
# OR
yarn add @yuno-payments/sdk-web-vtex
```

---

## Mount the SDK

Once installed, initialize and mount the SDK on the page where the shopper completes the payment.

```javascript
import { mount } from '@yuno-payments/sdk-web-vtex';

mount({
  elementRoot: 'yuno-sdk-root',
  payload: payloadFromConnector, // Provided by the Yuno Payment Connector
  language: 'en',
  domainVTEX: 'yourstore.myvtex.com',
  onPaymentDone: (result) => {
    // Continue to your order confirmation flow.
  },
  onError: (error) => {
    // Handle and surface the error to the shopper.
  },
  onLoading: (isLoading) => {
    // Show or hide a loading indicator.
  },
});
```

### Configuration fields

| Field | Description |
| :--- | :--- |
| `elementRoot` | The ID of the DOM element where the SDK should render the checkout UI. |
| `payload` | The payment context for the current shopper, provided by the Yuno Payment Connector. |
| `language` | The language to display the SDK in (e.g., `en`, `es`, `pt-BR`). Falls back to English if unsupported. |
| `domainVTEX` | Your VTEX store domain. |
| `onPaymentDone` | Callback invoked when the payment finishes. Receives the payment result. |
| `onError` | Callback invoked when an error occurs during the payment. |
| `onLoading` | Callback invoked when the SDK changes its loading state. |
