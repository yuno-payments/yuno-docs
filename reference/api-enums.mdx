This page serves as the **Single Source of Truth (SSOT)** for all key enums and identifiers across the Yuno platform. Consolidating these values here ensures consistency and provides a single location for developers to find valid parameters for API requests.

<AccordionGroup>
  <Accordion title="Core API Identifiers" icon="brackets-curly">
    Essential identifiers used for routing, processing, and integration strategy.

    ### provider_id
    Identifies the payment service provider (PSP) or gateway.
    [View full list of 120+ providers](#provider-ids)

    ### payment_method_type
    Specifies the payment instrument used.
    [View full list of 180+ methods](#payment-method-types)

    ### type_of_enrollment_flow
    Integration strategy for enrolling payment methods.

    | Value | Description |
    | :--- | :--- |
    | `SDK_FULL` | Full enrollment flow managed by Yuno SDK. |
    | `SDK_LITE` | Lite enrollment flow with partial UI control. |
    | `SDK_HEADLESS` | Headless enrollment flow for full UI customization. |
    | `DIRECT_API` | Enrollment via direct API calls (Server-to-Server). |
  </Accordion>
</AccordionGroup>
