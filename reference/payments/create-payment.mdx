This request creates a payment after you created a checkout session.

<Danger>
**Important**

Although certain objects are not mandatory when creating a **Payment**, the user’s payment experience can be enhanced if you provide this information.
</Danger>

<Note>
**3DS Payments**

Check the [3D Secure](/docs/security-and-compliance/3d-secure) page to learn how to create payments using 3DS on Yuno.
</Note>

If you want to process payments in a Single Step (Authorization and Capture simultaneously), send `payment_method.detail.card.capture` as `true`.

To test all possible transaction outcomes within Yuno in **Sandbox**, refer to the [test data documentation](/docs/direct-integration-use-cases/yuno-testing-gateway#4-provide-the-payment-method-information).

<Danger>
**Metadata and routing logic**

If you use metadata to drive routing logic, it must also be set in the **[Checkout Session](/reference/checkout-sessions/create-checkout-session)**. Filling it only in the Payment object will not activate route logic.
</Danger>

For Two-Step payments (Authorization then Capture), send `payment_method.detail.card.capture` as `false`, then use the [Capture Authorization](/reference/payments/capture-authorization) endpoint.

This request requires an `X-Idempotency-Key`. See [Authentication](/reference/authentication#idempotency) for details.

## Body

<ParamField body="account_id" type="string" required>
  The unique identifier of the account. Find this on [the Yuno dashboard](https://dashboard.y.uno/) (UUID, 36 chars).
</ParamField>

<ParamField body="merchant_order_id" type="string" required>
  The unique identifier of the customer’s order (MAX 255; MIN 3).
</ParamField>

<ParamField body="description" type="string" required>
  The description of the payment (MAX 255; MIN 3).
</ParamField>

<ParamField body="country" type="string" required>
  Country where the transaction must be processed (MAX 2; MIN 2; [ISO 3166-1](/reference/country-reference)).
</ParamField>

<ParamField body="amount" type="object" required>
  The payment amount object, with value and currency.

  <Expandable title="amount attributes">
    <ParamField body="currency" type="string" required>
      The currency used to make the payment. See [Country reference](/reference/country-reference) for the full list (MAX 3; MIN 3; ISO 4217).
    </ParamField>

    <ParamField body="value" type="number" required>
      The payment amount (multiple of 0.0001).
    </ParamField>

    <ParamField body="currency_conversion" type="object">
      The currency conversion (DCC) object. Used when the cardholder pays in their home currency while the merchant settles in their own. `cardholder_currency` must differ from `amount.currency`.

      <Expandable title="currency_conversion attributes">
        <ParamField body="id" type="string">
          Unique identifier of the conversion rate query from Yuno (UUID, 36 chars).
        </ParamField>
        <ParamField body="provider_currency_conversion_id" type="string">
          Unique identifier of the conversion rate query from the provider (MAX 526; MIN 3).
        </ParamField>
        <ParamField body="cardholder_currency" type="string">
          Currency the cardholder is charged in (ISO 4217 alpha-3). Must differ from `amount.currency`. Required if `currency_conversion` is included.
        </ParamField>
        <ParamField body="cardholder_amount" type="number">
          Total amount billed to the cardholder in `cardholder_currency` (multiple of 0.0001). Required if `currency_conversion` is included.
        </ParamField>
        <ParamField body="cardholder_accepted" type="boolean">
          Cardholder’s explicit consent to the DCC offer. `true` = accepted; `false` = declined.
        </ParamField>
        <ParamField body="rate" type="number">
          Applied FX rate (markup included). Must be greater than 0.
        </ParamField>
        <ParamField body="rate_margin_percentage" type="number">
          Markup over the wholesale rate, as a percentage (0–100).
        </ParamField>
        <ParamField body="rate_source" type="string">
          Wholesale benchmark used for the quote. Required when `rate_margin_percentage > 0`. Examples: `ECB`, `REUTERS`, `BLOOMBERG`.
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="payment_method" type="object" required>
  The payment method object.

  <Expandable title="payment_method attributes">
    <ParamField body="token" type="string">
      One-time-use payment method token **provided by Yuno SDK**. If sent, `vaulted_token` is not required (MAX 64; MIN 36).
    </ParamField>

    <ParamField body="vaulted_token" type="string">
      Token for a **previously stored payment method**. If sent, `token` is not required (MAX 64; MIN 36).
    </ParamField>

    <ParamField body="type" type="string">
      Type of the payment method. Required for `DIRECT` or `REDIRECT` workflow, and for wallets even when using the SDK.
    </ParamField>

    <ParamField body="vault_on_success" type="boolean">
      Enroll the card after a successful payment. Requires `customer_payer.id`.
    </ParamField>

    <ParamField body="detail" type="object">
      Payment method detail object. Contents vary by payment method type.
    </ParamField>

    <ParamField body="otp" type="object">
      OTP object. Used for payment methods that require one-time passwords.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="checkout" type="object" required>
  The checkout object. Not required for back-to-back payments. Required when `workflow` is `SDK_CHECKOUT` or not sent. Not required for `DIRECT` or `REDIRECT` payments.

  <Expandable title="checkout attributes">
    <ParamField body="session" type="string">
      The checkout session token.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="additional_data" type="object" required>
  Additional data to enhance the payment experience. Not mandatory, but improves fraud screening and payment routing.

  <Expandable title="additional_data attributes">
    <ParamField body="seller_details" type="object" required>
      Seller details object.
    </ParamField>

    <ParamField body="order" type="object">
      Order details object.
    </ParamField>

    <ParamField body="airline" type="object">
      Airline information object. Required for airline transactions.
    </ParamField>

    <ParamField body="transportations" type="array">
      Transportation details array of objects.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="workflow" type="string">
  The payment workflow type.

  Available options: `SDK_CHECKOUT` (recommended — uses Yuno SDK), `DIRECT` (back-to-back with provider info), `REDIRECT` (back-to-back with provider redirection).
</ParamField>

<ParamField body="callback_url" type="string">
  URL to redirect the customer after payment. Required only for `DIRECT` integrations with redirection (MAX 526; MIN 3).
</ParamField>

<ParamField body="merchant_reference" type="string">
  Merchant-defined identifier for the payment transaction (MAX 255; MIN 3). Optional complement to `merchant_order_id`.
</ParamField>

<ParamField body="customer_payer" type="object">
  Customer object for the payment.

  <Expandable title="customer_payer attributes">
    <ParamField body="id" type="string">
      Yuno customer UUID (36 chars). If sent, the rest of `customer_payer` is not required — the previously created customer’s data is used automatically.
    </ParamField>

    <ParamField body="merchant_customer_id" type="string">
      Merchant’s own customer identifier (MAX 255; MIN 1).
    </ParamField>

    <ParamField body="merchant_customer_created_at" type="string">
      Customer registration date on the merchant’s platform (ISO 8601, MAX 27; MIN 27).
    </ParamField>

    <ParamField body="first_name" type="string">
      Customer’s first name (MAX 255; MIN 1).
    </ParamField>

    <ParamField body="last_name" type="string">
      Customer’s last name (MAX 255; MIN 1).
    </ParamField>

    <ParamField body="email" type="string">
      Customer’s email address (MAX 255; MIN 1).
    </ParamField>

    <ParamField body="gender" type="string">
      Customer’s gender (MAX 2; MIN 1). Values: `M` (Male), `F` (Female), `NB` (Non Binary), `NA` (Not applicable), `NK` (Not Known), `U` (Undefined).
    </ParamField>

    <ParamField body="date_of_birth" type="string">
      Customer’s date of birth in `YYYY-MM-DD` format.
    </ParamField>

    <ParamField body="nationality" type="string">
      Customer’s nationality (MAX 2; MIN 2; [ISO 3166-1](/reference/country-reference)).
    </ParamField>

    <ParamField body="ip_address" type="string">
      Customer’s IP address (MAX 45; MIN 1).
    </ParamField>

    <ParamField body="phone" type="object">
      Customer’s phone number object.
    </ParamField>

    <ParamField body="document" type="object">
      Customer’s document object (number and type).
    </ParamField>

    <ParamField body="billing_address" type="object">
      Customer’s billing address object.
    </ParamField>

    <ParamField body="shipping_address" type="object">
      Customer’s shipping address object.
    </ParamField>

    <ParamField body="browser_info" type="object">
      Browser information object. Required for 3DS flows.
    </ParamField>

    <ParamField body="device_fingerprints" type="array">
      Device fingerprints (MAX 4000; MIN 1). Required for `DIRECT` integrations with fraud providers. For SDK integrations, obtained automatically — do not send this field.
    </ParamField>

    <ParamField body="geolocation" type="object">
      Customer’s geolocation object.
    </ParamField>

    <ParamField body="merchant_customer_validations" type="object">
      Fraud validation data for the customer payer.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="subscription" type="object">
  Subscription object. Use when creating a payment linked to a subscription.

  <Expandable title="subscription attributes">
    <ParamField body="id" type="string">
      Unique identifier of the subscription (used when creating a payment for an existing subscription).
    </ParamField>

    <ParamField body="frequency" type="object">
      Frequency object — defines billing cadence when creating a new subscription.
    </ParamField>

    <ParamField body="availability" type="object">
      Availability object — defines duration when creating a new subscription.
    </ParamField>

    <ParamField body="amount" type="object">
      Amount object for the subscription payment.
    </ParamField>

    <ParamField body="billing_date" type="object">
      Billing date object — used when creating a payment for an existing subscription.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="split_marketplace" type="array">
  Marketplace split array. Defines how the payment amount is distributed across recipients.

  <Expandable title="split_marketplace item attributes">
    <ParamField body="type" type="string" required>
      Split type. `recipient_id` is mandatory for `PURCHASE` and `MARKETPLACE`.

      Available options: `PURCHASE`, `PAYMENTFEE`, `VAT`, `COMMISSION`, `MARKETPLACE`, `SHIPPING`.
    </ParamField>

    <ParamField body="recipient_id" type="string">
      Yuno-generated recipient identifier. Provide either this or `provider_recipient_id`.
    </ParamField>

    <ParamField body="provider_recipient_id" type="string">
      Recipient ID from the external payment provider. Provide either this or `recipient_id`.
    </ParamField>

    <ParamField body="description" type="string">
      Description for the split (MAX 255; MIN 3).
    </ParamField>

    <ParamField body="merchant_reference" type="string">
      Optional unique identifier for the split transaction (MAX 255; MIN 3).
    </ParamField>

    <ParamField body="recipient_type" type="string">
      Type of recipient for the provider. Available options: `MEAL`, `FOOD`, `MULTI_BENEFITS`, `FLEET`.

    <ParamField body="amount" type="object">
      Split amount. Optional if the recipient has a `split_configuration` defined.

        <ParamField body="chargebacks" type="boolean">

        </ParamField>
      </Expandable>

  </Expandable>

  Fraud screening object.

      Stand-alone fraud screening configuration.
