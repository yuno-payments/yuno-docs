title: "Updates for March, 2025"

You can take a look at the latest Yuno updates. There have been some updates on Yuno. Please check the updates below.

* Discounts
* Monitors Flag
* Customer info in Refunds

## Discounts

We've added a new capability to the Payments API, which allows you to specify the information of the discounts related to the purchase.

```json
[...],
"additional_data": {
        "order": {
            "fee_amount": 40.5,
             "discounts": [
                {
                    "id": "XYZ",
                    "name": "Offer",
                    "unit_amount": "100"
                }
            ],
[...]
```

Please refer to the [payments](/reference/create-payment)  API reference and the [Payment amount details](/docs/payment-amount-details#discounts) section for more information.

## Monitors flag

We've added a new field to the [Payment object](/reference/the-payment-object) response called `monitors`. It's a field inside the routing rules object specifying whether a transaction went through the monitors feature.

```json Example
[...], 
"routing_rules": {
        "smart_routing": false,
        "monitors":true,
        "condition": {
            "id": 133892,
            "name": null,
            "description": null
        }
    },
[...]
```

For more information, please refer to the [payments](/reference/create-payment) API reference section.

## Customer info in refunds

We've added a new capability to the Payments API, which allows you to specify the customer information in the refund request in case you need to send additional data to the payment provider.

```json Example
{
    "description": "Duplicate",
    "reason": "REQUESTED_BY_CUSTOMER",
    "merchant_reference": "AAB01-432245", 
    "customer_payer": {
        "document": {
            "document_type": "DNI",
            "document_number": "38919283"
        }
    }
}
```

For more information, please refer to the [refunds](/reference/refund-payment) API reference section.
