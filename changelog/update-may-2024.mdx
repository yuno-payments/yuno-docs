There have been some updates on Payments object and transaction requests. Please check the updates below.

* Simplified mode
* Benefit Type

### Simplified mode

In Yuno we let merchants [retry capture](/docs/transaction-retries) transactions with an error response status just by setting an additional field in the [Capture request](/reference/capture-authorization) . The feature is designed to enhance transaction success rates and improve user experience. By setting the "simplified\_mode" field to true in capture requests, the system will automatically retry failed transactions up to 5 times within a 24-hour period.

*Example*

```json
{
  "amount": {
    "currency": "COP",
    "value": "5000"
  },
  "simplified_mode": true,
  "reason": "PRODUCT_CONFIRMED",
  "merchant_reference": "AA01"
}
```

### Benefity Type

We've added a new field to the [Payment object](/reference/the-payment-object) called `benefity_type`. It is a field that represents the user's benefity type for the payment method Pluxee for Chile. Necessary for DIRECT workflow.
