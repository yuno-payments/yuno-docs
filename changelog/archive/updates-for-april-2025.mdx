title: "Updates for April, 2025"
---

There have been some updates on the Direct implementation for the 3DS service. Please check the updates below.

### 3DS - Direct implementation

The [setup service](/docs/direct-integration-use-cases/3ds-configuration-and-testing) for the 3DS direct implementation is no longer required. We included the collection of data and setup handling inside the payment creation, making the implementation easier and faster. More information in the [3DS Secure guide section](/docs/security-and-compliance/3d-secure).

### Additional Airline data

We've added a new capability to the Payments API, which allows you to specify more information of the airline object related to the purchase.

```json
[...],
 "additional_data": {
        "airline": {
            "legs": [
                {
                    "arrival_airport": "AMS",
                    "arrival_airport_country": "AR",
                    "arrival_airport_city": "BUE",
                    "departure_airport": "EZE",
                    "departure_airport_country": "AR",
                    "departure_airport_city": "BUE"
                }
            ],
            "pnr": "1P-2UUGJW",
             "tickets": [
                    {
                    "issue": {
                        "booking_system_code": "TT01",
                        "booking_system_name": "booking system"
                    }
                    }
                ]
        },
[...]
```

Please refer to the [payments](/reference/payment)  API reference section for more information.
